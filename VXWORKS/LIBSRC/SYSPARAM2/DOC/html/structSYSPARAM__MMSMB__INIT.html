int32(* SYSPARAM_MMSMB_INIT::eeRead)(void* osh, int smbBus, int smbAdr, int offset, u_int8 *buf, int bufLen )
 

function callback to read eeproms,

Should read bufLen bytes from offset of the SMB eeprom at smbBus and smbAdr into buf. The data block may cross a 256 byte boundary.

This function is "missused" also for NVS sections in (boot) flash. If smbBus is SYSPARAM_SMB_BUS_FLASH, then the NVS section is in flash. In this case smbAdr is interpreted as a sector identifier, e.g. 0xff for last sector in flash. offset is measured from the beginning of addresses flash sector.

Returns:
0 if ok, a value != 0 on error

int32(* SYSPARAM_MMSMB_INIT::eeWrite)(void* osh, int smbBus, int smbAdr, int offset, u_int8 *buf, int bufLen )
 

function callback to write eeproms,

Should write bufLen bytes to offset of the SMB eeprom at smbBus and smbAdr from buf. The data block may cross a 256 byte boundary. The function should perform a read-after-write.

Also used for flash, see note in eeRead().

Returns:
0 if ok, a value != 0 on error

char* SYSPARAM_MMSMB_INIT::mmParaStringAdrs
 

MENMON parameter string address.

The content of this string is used to initialize the parameters in the mmpara parameter group (cpu=, pciclkhz=...). It should be therefore either be

  • initialized with parameters or
  • an empty string and the parameters updated later by firmware

This string is then filled by SYSPARAM with most of the parameters and whenever a SysParamSet() or SysParamSetDefaults() is issued.

char* SYSPARAM_MMSMB_INIT::vxBlineStringAdrs
 

VxWorks bootline string address.

This string is the immediate buffer for the parameters in the vxbline parameter group.

When it is an empty string when the xxxSysParamInit() routine is called, SYSPARAM reads the string from non volatile storage.

If it is a non-empty string, the string is basically validated.

Whenever either the NV data or the passed string appears invalid, a default bootline is setup (CPU dependant).

This string shall have space for at least 480 bytes.

vxBlineStringAdrs will receive immediately any modification made through SysParamSet() or SysParamSetDefaults().

The string can be also modified outside the SYSPARAM library. The SysParamGet() will fetch the values always directly from this string.
