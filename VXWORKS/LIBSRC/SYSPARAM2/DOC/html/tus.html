Definition of tuple strings

Tuple strings are used e.g. in the MENMON parameter string to store multiple parameters with their value in a single ASCII string.

The following does not apply to VxWorks bootline, which has a similar, but not identical format.

Example for a tuple string: 

 "par1=value1 par2='value with blanks' par4=yy"

  • Parameter names may consist of ASCII chars [0x21..0x7e], except blanks and '=' chars.
  • Parameter names are case sensitive
  • A parameter name must not appear more than once in the tuple string
  • The equal sign '=' must immediately follow the parameter name
  • The parameter value starts after the '=' and ends with the next blank or EOS.
  • If the value contains blanks, value must be surrounded by ticks '\''
  • Ticks can be nested, but only if each opening tick is preceeded by a '=' character, e.g: 
     "par2='value with blanks subpar='nested value''"

  • Parameter value may contain ASCII chars [0x20..0x7e]
  • Tuples are separated by any number of blanks
