enum SYSPARAM_ACC_ATTR
 

parameter access attributes

Returned in the SYSPARAM_PARAM_INFO structure by SysParamEnum()

(some flags can be binary ORed)

Enumeration values:
SYSPARAM_ACC_RO  read-only for user, writeable from firmware
SYSPARAM_ACC_RW  read/write
SYSPARAM_ACC_PW  read-only, writeable with password
SYSPARAM_ACC_RW_MASK  mask for above flags
SYSPARAM_ACC_ENCRYPT  parameter shall be displayed encrypted
SYSPARAM_ACC_NEEDRESET  parameter needs reset to take effect
SYSPARAM_ACC_MMPARASTR  parameter is part of MENMON parameter string
SYSPARAM_ACC_PERSISTANT  parameter can be saved non-volatile

enum SYSPARAM_SET_ATTR
 

attributes for SysParamSetEx (can be binary ORed)

Enumeration values:
SYSPARAM_SA_NON_PERSISTANT  don't immediately write to NVS
SYSPARAM_SA_FROM_FIRMWARE  attempt to set from firmware

int SYSP_MenBrdNameWithModel (  char  prefix,
char *  buf
) 
 

Determine chars 3..9 (e.g.

EM04-00) of MEN product name

Uses SYSP_brd and SYSP_brdmod parameter. SYSP_brd value is padded with '-' if it contains only 4 chars

If SYSP_brd not valid, return empty string. If SYSP_brdmod not valid, model number will be missing from string

Parameters:
prefix IN board prefix (e.g. 'c') (0 if none)
buf OUT filled with name (must have space for 7 chars + EOS)
Returns:
sysparam error code

int SysParamGet (  const char *  parName,

int  parValMaxLen
) 
 

Get value of single system parameter.

This gets the current value of the parameter parName. Parameter name can be either the official parameter name or the alias name.

If SysParamGet() returns non-zero, parVal will contain an empty string (unless parValMaxLen is 0).

If the parameter value (as a string) is longer than parValMaxLen, string in parVal is truncated. parVal is always guaranteed to be null-terminated.

parVal may contain blanks.

This function can be used to probe for the existance of a parameter name. Pass parVal as NULL and parValMaxLen as 0 in this case.

Parameters:
parName IN parameter name including possible prefix.
parVal OUT receives the parameter's value as a string
parValMaxLen IN maximum bytes that parVal can hold, including the terminating '\0'
Returns:
0 on success, or sysparam error code
  • SYSPARAM_NOT_INIT: xxxSysParamInit() has not been called
  • SYSPARAM_INVAL_PARAM: parameter is not known
  • SYSPARAM_INVAL_VALUE: parValMaxLen is 0 and parVal is not NULL. Or internal representation is damaged
  • SYSPARAM_VAL_TRUNCATED: only a part of value could be copied to parVal (and parVal was not NULL)

int SysParamSetEx (  const char *  parName,
const char *  parVal,
const char *  passwd,
int  attr
) 
 

Modify value of single system parameter (extended version).

This updates the internal representation of the specified parameter parName. Parameter name can be either the official parameter name or the alias name.

If SYSPARAM_SA_NON_PERSISTANT is not specified in attr, the parameter is also written into non volatile storage (NVS).

If SYSPARAM_SA_NON_PERSISTANT is given, NVS is not updated. NVS can be updated by another call to SysParamSetEx for a parameter in the same parameter group (with SYSPARAM_SA_NON_PERSISTANT clear) or by calling SysParamMakePersistant().

Parameters that are read-only for the normal user can be written to by specifying attribute SYSPARAM_SA_FROM_FIRMWARE in attr. This flag should be set only when called from firmware.

A parameter's value can be cleared by passing an empty string ("") in parVal.

Restart when production data name has changed When modifying the name of a production data section ("brd" parameter), the SYSPARAM library is entirely restarted, since the new name may imply a new EEPROM layout. After SYSPARAM has been restart (which is invisible to the user) the available parameters may have changed. Modifying the board name is only possible with SYSPARAM_SA_NON_PERSISTANT flag cleared.

Parameters:
parName IN parameter name including possible prefix.
parVal IN new value for parameter as a string
passwd IN an optional password for protected parameters can be NULL if parameter is not protected
attr IN attribute flags (see SYSPARAM_SET_ATTR)
Returns:
0 on success, or sysparam error code
  • SYSPARAM_NOT_INIT: xxxSysParamInit() has not been called
  • SYSPARAM_INVAL_PARAM: parameter is not known
  • SYSPARAM_INVAL_VALUE: invalid value for parameter or parVal too long
  • SYSPARAM_EEPROM_ERROR: Error during NVS update
  • SYSPARAM_READ_ONLY: Parameter is read only and SYSPARAM_SA_FROM_FIRMWARE was not set in attr, or bad password was specified.
  • SYSPARAM_NO_RESOURCE: parVal could not be stored due to lack of space (in NV storage or temp. buffers).
  • SYSPARAM_VAL_TRUNCATED: only parts of parVal have been stored.
